1) Purpose and Scope

  • Document complete system behavior for external engineering teams.
  • Cover architecture, business flows, technical contracts, and schema.
  • Explain verification/OTP/payment controls and security boundaries.
  • Provide operational guidance and known caveats for safer changes.

2) Stack and Runtime

Core

  • Framework: Laravel (PHP)
  • Rendering: Blade server-side templates
  • DB Access: Query Builder + direct DB operations
  • Logs: storage/logs/laravel.log

Operational

  • Background command: php artisan emd:process-defaults
  • Payment callback endpoints with signature validation
  • Session middleware: session.auth, session.admin
  • Mixed schema source: migrations + legacy DB tables

6) Integration Contracts

6.1 PayU

  • Used for registration payment, auction payment, and participation fee flow.
  • Callback signature/hash is verified before finalizing transaction state.
  • Callback endpoints exist under /payu/* and are webhook-compatible.

6.2 SMS Gateway (MyToday BulkPush)

  • Service: app/Services/BulkSmsService.php
  • Endpoint default: https://bulkpush.mytoday.com/BulkSms/SingleMsgApi
  • Request sent as application/x-www-form-urlencoded.
  • MSISDN formatting: sends Indian numbers with 91 prefix.
  • Optional params: entityid, async (if configured).
Config KeyPurpose
SMS_BULK_ENABLEDEnable/disable real SMS send
SMS_BULK_URLGateway endpoint
SMS_BULK_FEED_IDGateway account feed id
SMS_BULK_USERNAMEGateway username
SMS_BULK_PASSWORDGateway password
SMS_BULK_TEMPLATE_IDDLT-approved template id
SMS_BULK_SENDER_IDSender id
SMS_BULK_ENTITY_IDOptional DLT entity id
SMS_BULK_ASYNCOptional gateway mode

6.3 SMTP / Email

  • Used in registration, profile OTP email, admin messages, payment notifications.
  • Supports dynamic SMTP settings from DB (email_settings) in existing implementation paths.

6.4 PAN Verification (IdFy)

  • Registration calls IdFy APIs for PAN verification task creation and status checks.
  • Result gates continuation of registration flow.

9) Security and Verification Controls

OTP Policy

  • 6-digit OTP, TTL 10 minutes.
  • Stored in session with scoped keys by user id/hash target identity.
  • Invalid/expired OTP blocks state transition.

Blacklist and Fraud Controls

  • Identity vectors: email, mobile, PAN, device fingerprint, IP.
  • Applied in registration, login, and profile identity changes.

Payment Trust Boundary

  • Gateway callbacks validated by hash/signature.
  • Duplicate callback handling guards against repeat updates.

Auditability

  • audit_logs for security/operations events.
  • bid_logs for bid lifecycle visibility.
  • user_identity_change_logs for profile update traceability.

10) Operations Runbook

10.1 Environment and config checklist

  • DB connectivity + migration consistency
  • PayU credentials + callback URLs
  • SMTP settings (env or DB-configured email settings)
  • SMS gateway variables and DLT mappings

10.2 Post-config command

php artisan config:clear

10.3 Scheduled command

php artisan emd:process-defaults - promote next bidder when payment window expires.

10.4 Incident troubleshooting pointers

  • SMS not delivered: inspect BulkSms logs + provider TID delivery dashboard.
  • Payment mismatch: verify callback hash and transaction reference mapping.
  • Profile OTP issue: inspect session key lifetimes and profile pending state keys.

11) Known Caveats and Risks

  • Core table DDL not fully represented in migrations (portability risk).
  • Custom session auth pattern may be misread as standard guard flow.
  • Wallet/EMD features exist in code but several routes are intentionally disabled.
  • Debug/non-production OTP exposure paths must remain restricted in prod.
  • Runtime table/config assumptions should be normalized into migration-driven infra over time.

12) New Team Handoff Checklist

  1. Export and version all legacy table DDL from production/staging.
  2. Create ERD and confirm all FK/logical relationships.
  3. Validate all route groups, middleware, and role paths.
  4. Run complete OTP test suite: registration, profile, forgot-password.
  5. Run payment callback end-to-end staging tests with gateway signatures.
  6. Validate blacklist behavior across identity vectors.
  7. Validate invoice generation and storage permissions.
  8. Enable and monitor scheduled emd:process-defaults execution.
